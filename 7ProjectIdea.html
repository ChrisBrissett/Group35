Project Description

Overview

After receiving feedback from Assignment 2, our group is determined to start work on what is remarked as being a unique and easily expandable idea. Through deep planning and market research, we have begun the build of a new app, an app that will mitigate lack of score sheets across a range of board games. Our first target is for the game Cluedo. Our team are working hard towards delivering a fully functional app that takes away the need for physical score cards. This app is driven to keep the game rolling, without taking away the social dynamic of playing a board game with your friends or family.

As you can appreciate, there are thousands upon thousands of board games available. With our project open ended, we plan to add new games as they are released and continue to digitise the classic board game experience.

Not only is this project viable in the real-world market, it also follows the current IT trend of cloud computing (2021 IT (Information Technology) Industry Trends Analysis | Business of Technology | CompTIA 2020) a and provides our members valuable experience and opportunity to develop key skills needed for our ideal careers. Cloud computing for example is a key skill required for both Paul and Kayla, as System Engineer, and Cyber Security engineer as their respective career choices. This project also gives Rich an opportunity to show experience managerial qualities, and Bernadette will be able to develop marketing strategies and develop a portfolio. As a prospective game programmer, Chris will find many opportunities to develop his coding and software knowledge.

Topic

Our project idea is to develop and implement an app for use on the Google Play Store and Apple App Store, which would replace the score sheets and clue cards when playing the board game Cluedo. The game will be able to keep track of each game you play, making it a more effective method of being able to keep track of what cards you were dealt along with being able to make off options you have ruled out in the current game.

The ideal outcome for the app would be to have a functional replacement for Cluedo score cards and clue-keeping sheets initially. We will then look at developing further functions such as random card shuffling and distribution amongst players, as well as storing the case file cards.

Motvation

The motivation behind this idea is that many people worldwide who still enjoy the social experience of playing board games with family and friends, is that they find that pieces of the game constantly go missing. The biggest issue specifically for the game of Cluedo is having to re-print pages upon pages of scoresheets to replace the sheets already consumed. Nowadays, board games have been uploaded to the internet allowing a much larger opportunity of players, trying to combine the useability of mobile phones and the social togetherness of playing board games is the driving motivation behind the idea. By doing this and creating the app, we are looking at following trends of having a companion app for board games to incorporate the use of mobile phones. Many game titles played on PC or console have a companion app used to track transfer and monitor game progress. What we aim to do is provide a similar service to a different demographic of users.

Landscape

While researching the current environment for boardgame apps and other score keeping apps. There seems to only be a select few apps of quality and popularity. In regards to Clue/do specific apps there is one that I could find, ‘Clue Solver - for Clue or Cluedo board games’, from publisher Brady Anderson on the app store. Released in 2012, the only reviews seem to be negative stating the app was buggy or freezes often. With no updates in last 8 years I would say it is not much in the way of competition. The only other app I found similar to our own and with any sort of popularity is a game score keeping app called ‘ScorePal’ published by HappyFaceDevs on the Google Play store. The last update to this game was in 2018; it also has positive and recent reviews by users. Our advantage over this app would be our in-game mechanic support (such as card management and clue keeping).

After investigating the current landscape, we have devised a new goal, which is to provide regular and consistent updates. Doing this should provide ample incentive to new users, compared to the competition and the lack of recent developments. With the current environment reasonably untouched, should we release this app we believe there would be little to no competition.

Tools and Technologies

Code/App Development

After review of feedback given and extensive research into finding a suitable cross platform developer, we have decided to use Microsoft’s Visual Studio version 16.8.1, Android.Sdk.Manager version 4.1.1 & Xamarin version 4.8.0 [1]. With this we will be provided with a single, cross platform API (Application Programming Interface) to develop code to be used across iOS and Android [2] using C# as the language.

Xaramin extends .NET with tools and libraries designed specifically for building apps on iOS & Android. Microsoft’s Visual Studio and Xaramin are free and require no licensing costs. We will use these development tools along with GitHub to opensource and code we may require.

To use this we will need access to a Desktop computer or laptop with access to the internet. As the developer tools we will be using can be used on either Windows or macOS there will be no requirement to use a dedicated operating system.

Storage/Connectivity

To store user information and records we will use Amazon Web Services (AWS) Mobile app Backend [3] to access Amazon Cognito User Pools. This will allow users of the app to sign up and sign in to the application. Here is a diagram from Amazon depicting their cloud database structure.

We will use AWS GameLift to manage the servers and allow interconnectivity of the game. As our app requires interaction between multiple devices in real-time, this is a core component to ensuring the viability of the product. We aim to achieve this by using the Amazon GameLift SDK Version 4.0.2 [4]. Below is a basic database diagram from AWS GameLift.

As we are using a cloud-based solution to store and connect users we do not need any physical hardware to achieve this. We have, however, considered that we will need our own database to store on AWS we designed a very basic data structure of our own. This covers our most necessary information such as user IDs, game IDs, account data, and history.
